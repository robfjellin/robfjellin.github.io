Professional Self-Assessment

Prior to starting my degree at Southern New Hampshire University I had little experience with software engineering, mainly in web development. Throughout the program I have taken many courses that have helped develop my software engineering skills, transforming me from someone with basic web development knowledge into a well-rounded software engineer that is now prepared to tackle real challenges.

My coursework has fundamentally given me a great understanding of what it means to collaborate within software development. In CS-250 (Software Development Lifecycle), I learned about agile frameworks and how to work within them, along with the importance of sprint planning, daily stand-ups, and retrospectives. This was one of the courses that taught me that successful collaboration with a team isn't just about writing code. It's about clear communication, accountability, and iterative improvement. In addition to that, IDS-105 (Cultural Awareness and Online Learning) helped me gain skills in navigating personal conflicts and really understanding how cultural diversity can impact a team's dynamics. These experiences have prepared me to contribute to diverse development teams, where I can balance my own contributions with collective team goals.

Throughout my program here at Southern New Hampshire University, I have developed the ability to break down complex concepts for a variety of audiences. CS-255 (System Analysis and Design) was particularly important, as one of the projects was to create a system model while being able to communicate the design decisions to diverse stakeholder groups. In CS-465 (Full Stack Development I), I took part in implementing end-user, stakeholder, and developer interfaces that aligned with the business objectives. This particular skill set is more than just documentation, rather, ensuring the stakeholders were really able to understand the rationale behind the design decisions.

My foundation in data structures and algorithms has been a vital part of development for me as a software engineer. CS-300 (DSA: Analysis and Design) taught me to evaluate the algorithmic complexity and choosing which data structure was best for a specific problem. During that course I learned that hash maps and binary search trees are more than just learning topics, but instead served their own real world purposes. MAT-230 (Discrete Mathematics) provided me with a theoretical foundation for understanding graph algorithms and logical reasoning which are very important for algorithmic problem solving. These courses taught me to balance competing concerns, whether it's optimization between speed and memory, or simple solutions versus scalable and flexible architectures.

My computer science coursework has emphasized the entire software development lifecycle, not just coding. CS-320 (Software Test, Automation QA) taught me that quality assurance should be at the forefront of development, not the end. Writing unit tests, understanding validation versus verification, and implementing continuous integration has made me a more disciplined engineer. DAD-220 (Introduction to Structured Database Environments) along with CS-340 (Client/Server Development) provided skills that complemented each other in database design and integration. I learned how to normalize databases, write queries, and create interfaces between applications and data. These courses really emphasized that data persistence and retrieval are fundamental to every modern application.

CS-305 (Software Security) changed how I approach software development. I learned during this class that you can't just implement security at the end, instead, it must be thought about and worked on during every stage of development. From basic input validation and encryption to secure authentication protocols and protection from common vulnerabilities like SQL injection or cross-site scripting, this course taught me to not only think like a developer but to put myself in the shoes of the attacker as well. This mindset has stuck with me and has applied to every project since. Rather than just focusing on the functionality, I am now always searching for potential security implications.

The primary artifact of my ePortfolio is a weight tracking application I developed during my CS-360 (Mobile Architect & Programming) class which demonstrates my ability to design, develop, and strategically enhance a mobile application. This project showcases many technical competencies developed throughout my program: mobile application development, database management, user interface design, data visualization, and algorithmic problem solving. The original weight tracking application provided functionality for users to log and monitor their weight entries on a fundamental level. Throughout my enhancement process I significantly upgraded the application's capabilities by implementing data visualization using MPAndroidChart. This enhancement retrieved and transformed raw data from the SQLite database and turned it into an interactive and meaningful graph that now allows users to identify trends and track progress on their weight loss journey.

Original Artifact

Weight Loss Tracker

The purpose of this application is to help users track their weight over time, providing them a solution for health and fitness monitoring. The mobile app allows users to log their daily weight measurements, visualize their progress through graphs/charts, and set a personalized weight goal. This was built using Android Studio with Java as the primary programming language. SQLite is used for local data storage, ensuring that the user's data is stored securely while also allowing for offline use.

This project demonstrates my skills in mobile appliation development, database management, and user intrerface design principles. This makes it an ideal artifact for showcasing my comptentencies in Software Design & Engineering, Algorithms & Data Structures, and Databases.
